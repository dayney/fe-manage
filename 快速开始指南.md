# AI培训快速开始指南

> 5分钟准备，立即开始培训！

---

## 🎯 培训前准备（5分钟）

### 1. 准备演示环境（2分钟）
- [ ] 安装Cursor（如果还没有）
- [ ] 准备一个真实的项目代码
- [ ] 准备几个可以演示的场景：
  - 一个需要重构的组件
  - 一个需要修复的Bug
  - 一个需要优化的性能问题

### 2. 准备开场演示（2分钟）
- [ ] 选择一个简单的代码生成场景
- [ ] 测试一下演示流程
- [ ] 准备数据对比（效率提升）

### 3. 准备材料（1分钟）
- [ ] 打开培训方案文档
- [ ] 打开Prompt模板库
- [ ] 准备工具安装指南链接

---

## 🚀 培训流程（2小时）

### 阶段一：震撼开场（30分钟）

#### 1. 开场白（5分钟）
```
"大家好，今天不是要教大家AI技术，
而是要让大家看到AI如何让我们的工作更轻松、更有趣。

我先问一个问题：你们有没有遇到过这样的情况？
- 写一个组件要花2小时，但逻辑其实很简单
- 遇到一个Bug，查了半天文档才解决
- 要写测试用例，但不知道从哪里开始

今天，我要告诉大家，这些问题，AI可以在几分钟内帮你解决。
而且，这不是未来，是现在。今天培训结束后，你们就能立即在工作中使用。

准备好了吗？让我们开始吧！"
```

#### 2. 现场演示（20分钟）

**演示1：5分钟生成完整组件（5分钟）**
```
1. 打开Cursor
2. 创建新文件
3. 输入Prompt：
   "生成一个带搜索、筛选、分页的表格组件，
   使用React + TypeScript + Tailwind CSS"
4. 展示生成的代码
5. 运行并展示效果
```

**演示2：3分钟修复Bug（5分钟）**
```
1. 打开一个有Bug的代码文件
2. 复制错误信息
3. 在Cursor中问："这个错误是什么原因？如何修复？"
4. 展示AI的分析和解决方案
5. 应用修复并验证
```

**演示3：2分钟优化性能（5分钟）**
```
1. 打开一个性能有问题的组件
2. 选中代码
3. Cmd/Ctrl + K，输入："优化这个组件的性能"
4. 展示优化后的代码
5. 解释优化点
```

**演示4：1分钟生成测试（5分钟）**
```
1. 选中一个函数
2. 问："为这个函数生成单元测试"
3. 展示生成的测试代码
4. 运行测试验证
```

#### 3. 数据对比（5分钟）
```
展示效率提升数据：
- 写组件：2小时 → 20分钟（6倍）
- 修复Bug：1小时 → 10分钟（6倍）
- 写测试：30分钟 → 5分钟（6倍）
- 代码审查：1小时 → 15分钟（4倍）
```

---

### 阶段二：立即上手（1小时）

#### 1. 工具安装（10分钟）
- 分发工具安装指南
- 指导安装Cursor
- 演示基本操作

#### 2. 5个必学场景（40分钟）

**场景1：代码生成（10分钟）**
```
演示：
1. 打开Cursor
2. 创建新文件
3. 输入详细的Prompt
4. 展示生成的代码
5. 解释Prompt的重要性

练习：
让每个人尝试生成一个简单组件
```

**场景2：代码解释（5分钟）**
```
演示：
1. 选中一段复杂代码
2. 右键 → "Explain"
3. 展示解释结果

练习：
让每个人解释自己项目中的一段代码
```

**场景3：Bug修复（10分钟）**
```
演示：
1. 准备一个Bug
2. 粘贴错误信息
3. 问AI如何修复
4. 应用修复

练习：
让每个人找一个Bug，用AI修复
```

**场景4：代码重构（10分钟）**
```
演示：
1. 选中一段代码
2. Cmd/Ctrl + K
3. 输入："重构这段代码，提高可读性"
4. 展示重构结果

练习：
让每个人重构一段自己的代码
```

**场景5：写文档（5分钟）**
```
演示：
1. 选中一个函数
2. 问："为这个函数写JSDoc注释"
3. 展示生成的注释

练习：
让每个人为一个函数生成注释
```

#### 3. 实战练习（10分钟）
```
让每个人：
1. 打开自己的项目
2. 选择一个实际任务
3. 用AI完成
4. 分享结果
```

---

### 阶段三：效率提升技巧（30分钟）

#### 1. Prompt工程（15分钟）
```
讲解：
1. 好的Prompt公式：角色 + 任务 + 上下文 + 输出格式
2. 对比好的Prompt和差的Prompt
3. 提供Prompt模板库
4. 演示如何使用模板
```

#### 2. 上下文管理（10分钟）
```
讲解：
1. 提供足够的上下文
2. 使用@符号引用文件
3. 分段处理大项目
4. 演示实际案例
```

#### 3. 迭代优化（5分钟）
```
讲解：
1. 第一轮：生成基础代码
2. 第二轮：添加功能
3. 第三轮：优化
4. 第四轮：完善
```

---

### 阶段四：激发探索（20分钟）

#### 1. 展示AI的可能性（10分钟）
```
展示：
1. AI能做什么（代码生成、优化、测试等）
2. AI的局限性（需要验证、不能替代思考）
3. 实际应用案例
```

#### 2. 挑战任务（5分钟）
```
提出挑战：
1. 效率挑战：用AI在30分钟内完成一个完整页面
2. 学习挑战：用AI学习一个新的库
3. 创新挑战：用AI设计一个工具函数库
```

#### 3. 建立学习社区（5分钟）
```
建立：
1. 学习群/文档
2. 案例分享机制
3. 定期分享会
4. Prompt模板库
```

---

```
回答：
1. 工具使用问题

3. 实际应用问题
4. 后续学习建议
```
