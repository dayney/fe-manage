### PPT 大纲（1110 版）

- 第一页：主讲人介绍
  - 姓名与头衔
  - 背景与专长（AI 前端 / RAG / 多模态）
  - 代表项目/成果（2-3 个）
  - 联系方式（邮箱/微信/二维码）

- 第二页：主讲内容目录（3 个大标题）
  - 一、团队目标与学习路径（面向前端）
    - 目标产出、覆盖场景、学习节奏、评估方式
  - 二、核心工作流与工具（以 Cursor 为主）
    - Cursor 工作流、Prompt 工程、代码生成/重构/测试、RAG/文档上下文、评测
  - 三、实战演示与团队落地
    - 现场实操、规范与模板、评审流程、行动计划

- 第三页起：一、团队目标与学习路径（内容页）
  - 目标产出：1 个业务相关 Demo + 团队 Prompt/工作流模板
  - 覆盖场景：新页面搭建、重构提效、接口联调、Bug 定位、单测生成
  - 学习路径：概念与范式 → 工具与工作流 → 实战与评测 → 团队落地
  - 评估方式：任务清单完成度、PR 质量、迭代速度与回归稳定性

- 下一页：二、核心工作流与工具（内容页）
  - Cursor 工作流：Chat → Composer → Edits → Diff/Review → Commit 信息生成
  - Prompt 工程：角色/约束/少样本、结构化输出、代码上下文拼接
  - 代码提效：需求到页面骨架、组件抽象、重构建议、单测/用例生成
  - 文档与 RAG：项目文档/接口定义/读我手册接入，检索与引用
  - 质量评测：Prompt 回归集、lint/typecheck/测试流水线的 AI 集成

- 下一页：三、实战演示与团队落地（内容页）
  - 现场实操 1：从需求到页面（设计 → 组件 → 状态 → 接口）
  - 现场实操 2：重构与可维护性（抽象、命名、拆分、复杂度控制）
  - 现场实操 3：联调与排错（日志、网络、边界、回归用例）
  - 团队落地：仓库约定（目录/注释/提示词）、PR 评审清单、协作规范
  - 行动计划：本周模板接入 → 下周试点项目 → 月内评审与优化闭环

- 倒数第二页：Q&A
  - 收集问题、现场解答、补充材料

- 最后一页：感谢倾听
  - 关键词复盘、二维码（课件/交流群）、反馈问卷链接
