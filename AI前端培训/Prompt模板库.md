## 🎯 使用技巧

### 1. 组合使用
```
可以组合多个模板，比如：
"优化这段代码的性能，并添加单元测试"
```

### 2. 迭代优化
```
第一轮：生成基础代码
第二轮：添加功能/优化
第三轮：修复问题
第四轮：完善细节
```

### 3. 提供上下文
```
在Cursor中使用@符号引用：
@文件名 - 引用整个文件
@函数名 - 引用函数
@类型名 - 引用类型定义
```

### 4. 明确约束
```
明确说明：
- 不能做什么
- 必须遵循什么规范
- 性能要求
- 兼容性要求
```

## 📝 模板使用示例

### 示例1：生成一个搜索组件
```
你是一个资深React开发者。
生成一个SearchInput组件，要求：
- 使用TypeScript
- 使用函数式组件 + Hooks
- Props类型定义完整
- 包含防抖功能（300ms）
- 支持清除按钮
- Loading状态
- 使用Tailwind CSS样式
- 支持响应式布局
- 添加JSDoc注释

Props：
- value: string
- onChange: (value: string) => void
- placeholder?: string
- onSearch?: (value: string) => void
- loading?: boolean

输出格式：
1. 完整代码
2. 使用示例
3. Props说明
```

### 示例2：优化性能
```
优化这个组件的性能：
[组件代码]

关注点：
- 避免不必要的重渲染（使用React.memo）
- 使用useMemo缓存计算结果
- 使用useCallback缓存函数
- 优化列表渲染（虚拟滚动）

保持原有功能不变。
输出优化后的代码和说明。
```
